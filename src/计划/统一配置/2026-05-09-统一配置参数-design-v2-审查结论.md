## §3 测试可执行性细节（M4 / M5）

### M4（Low）：V5 / V6 缺 `delenv("MODEL_API_KEY")` 显式声明

**当前 V5 描述**：
> 仅设 `MODEL_API_URL` + `MODEL_NAME` 时...
> `cfg.agent.api_key == "EMPTY"`（MODEL_API_KEY 未设时）

测试 fixture 通常需要显式 `monkeypatch.delenv("MODEL_API_KEY",
raising=False)`，否则继承外部 shell 注入的真实 key 时会让 `assert ==
"EMPTY"` 失败（CI 上更明显）。这是 v1 现有测试的标准做法（见
`@.../tests/test_submission_config.py:22`）。

**修订建议**：V5 / V6 测试在 setup 段补：
```python
monkeypatch.setenv("MODEL_API_URL", "http://internal.model/v1")
monkeypatch.setenv("MODEL_NAME", "test-model")
monkeypatch.delenv("MODEL_API_KEY", raising=False)
```

### M5（Medium）：V7 "不传 `llm` 时保持 `RunConfig().task_timeout_seconds`" 断言可执行性

**问题**：`submission.run(llm=None, ...)` 会走真实 `runner.run_single_task`
路径并尝试构造 `ChatOpenAI`（依赖 `MODEL_API_URL` / `MODEL_NAME`），
600s timeout 在 CI 上不可接受。

**修订建议**：V7 应明确标注 monkeypatch 路径：

```python
def test_default_run_uses_runconfig_defaults_without_llm(tmp_path, monkeypatch):
    """V7：不传 llm 时 summary['task_timeout_seconds'] 走 RunConfig 默认。"""
    from data_agent_langchain import submission
    from data_agent_langchain.config import RunConfig
    from data_agent_langchain.run.runner import TaskRunArtifacts

    monkeypatch.setenv("MODEL_API_URL", "http://internal.model/v1")
    monkeypatch.setenv("MODEL_NAME", "test-model")
    monkeypatch.delenv("MODEL_API_KEY", raising=False)

    # 关键：monkeypatch _run_single_task_impl 避开真实 LLM 构造与 600s 超时
    def _fake_impl(*, task_id, config, run_output_dir, llm, graph_mode):
        ...  # 同 test_per_task_failure_does_not_block_other_tasks 模板

    monkeypatch.setattr(submission, "_run_single_task_impl", _fake_impl)
    dirs = _submission_dirs(tmp_path)
    dirs["input_dir"].mkdir()
    _make_task(dirs["input_dir"], "task_1")
    _make_caps(dirs["gateway_caps_path"])

    summary = submission.run(
        **dirs,
        register_signals=False,
        llm=None,  # 关键：不传 llm 走 RunConfig 默认 timeout
        action_mode_override="json_action",
    )

    assert summary["task_timeout_seconds"] == RunConfig().task_timeout_seconds
    assert summary["max_workers"] == RunConfig().max_workers
```

**对偶分支** `llm is not None` 时 `task_timeout_seconds == 0` 已被
现有 `test_caps_file_with_tool_calling_true_proceeds` 等多条测试隐式
覆盖（`task_timeout_seconds=0` 走 in-process 不超时）；如要显式断言，
可在同测试加 `summary["task_timeout_seconds"] == 0` 一行。

**为什么标 Medium**：V7 的 `不传 llm` 分支是 v2 新增的**独有**保护，
若实施时漏 monkeypatch 会让 CI 卡 10 分钟（5 worker × 600s 实际短路
后仍要等 ThreadPool shutdown）。
