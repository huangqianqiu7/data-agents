## §1 背景与目标

### 1.1 用户诉求

> 让 `data_agent_langchain` 项目除了 `api_key` 和对应的 URL 以外的其他
> 参数，在本地运行和容器运行都是同一套，所有参数配置都集中在同一个
> 地方。

### 1.2 评测规则约束

`rules.zh.md` §5.2 / §5.4：LLM URL / Key **必须**通过环境变量注入，
**不得**硬编码到 YAML 或镜像中。本设计在该约束基础上把“非 LLM 身份”
字段也收敛到唯一权威。

### 1.3 目标

- 以 `src/data_agent_langchain/config.py` 中的 dataclass 默认值为
  **唯一权威**。
- 本地路径（`load_app_config`）与容器路径（`build_submission_config`）
  对“非身份、非容器特化路径”的所有字段产出的最终值必须相等。
- `submission.py` 不再包含可能与 dataclass 默认漂移的常量。

## §2 现状盘点

### 2.1 容器路径（`submission.build_submission_config`）

当前在 `submission.py` 中显式写死：

| 现状 | 位置 | 问题 |
| --- | --- | --- |
| `DEFAULT_MAX_WORKERS = 5` | 顶部常量 | 与 `RunConfig.max_workers = 5` 重复，易漂移 |
| `DEFAULT_TASK_TIMEOUT_SECONDS = 600` | 顶部常量 | 与 `RunConfig.task_timeout_seconds = 600` 重复 |
| `DEFAULT_MODEL_NAME = "qwen3.5-35b-a3b"` | 顶部常量 | 与 `AgentConfig.model = "gpt-4.1-mini"` 不一致 |
| `DABENCH_MAX_WORKERS` env 覆盖 | `_int_from_env` | 让容器值有可能偏离本地默认 |
| `DABENCH_TASK_TIMEOUT_SECONDS` env 覆盖 | `_int_from_env` | 同上 |
| 显式 `agent.action_mode="tool_calling"` | `build_submission_config` | dataclass 默认已为 `tool_calling`，重复 |
| 显式 `evaluation=EvaluationConfig(reproducible=False)` | `build_submission_config` | dataclass 默认已为 False，重复 |
| 显式 `tools=ToolsConfig()` | `build_submission_config` | 空构造，等价于不传 |
| 显式 `observability.langsmith_enabled=False` | `build_submission_config` | dataclass 默认已为 False，重复（`run()` 内还有 `replace` 双保险） |

### 2.2 本地路径（`load_app_config`）

- 从 `configs/local.yaml` 加载 → `_resolve_config_paths` → `_overlay_agent_env_vars`
  （`MODEL_NAME` / `MODEL_API_URL` / `MODEL_API_KEY` env 兜底
  `agent.model/api_base/api_key`）。
- 其他字段 YAML 缺省时直接走 dataclass 默认。
- 与容器路径的差异点全部集中在 §2.1 的列表中。

## §3 设计原则

1. **唯一权威**：`config.py` 的 dataclass 默认值是所有共享参数的唯一
   权威；任何代码路径都不得在“构造 `AppConfig`”时硬编码与默认值数值
   相同（或不同）的字面量。
2. **最小覆写**：`build_submission_config` 仅显式覆写 “容器特化的
   3 条路径 + LLM 三字段（model/api_base/api_key）”，其他全部交由
   `AppConfig` 的 `default_factory` 与各 dataclass 默认产生。
3. **env 协议唯一**：MODEL_API_URL / MODEL_API_KEY / MODEL_NAME 为
   本地与容器共用的 env 协议；其他 env（如 `DABENCH_*`）一律删除。
4. **测试等价替换**：删除任何测试用例必须给出等价替换，证明被删除
   的语义已由其他用例覆盖。

## §4 变更项

### 4.1 `src/data_agent_langchain/submission.py`

**删除**：

- 顶部常量：`DEFAULT_MAX_WORKERS = 5`、`DEFAULT_TASK_TIMEOUT_SECONDS = 600`、
  `DEFAULT_MODEL_NAME = "qwen3.5-35b-a3b"`。
- 辅助函数 `_int_from_env`。
- `__all__` 中对应导出项。

**修改 `build_submission_config()`**：

- 不再读取 `DABENCH_MAX_WORKERS` / `DABENCH_TASK_TIMEOUT_SECONDS`。
- `MODEL_API_URL` 缺失抛 `SubmissionConfigError`（保留）。
- `MODEL_API_KEY` 缺失回退 `EMPTY_API_KEY`（保留）。
- `MODEL_NAME` 缺失：**不向 `AgentConfig(...)` 显式传 `model=`**，让
  dataclass 默认接管。
- 删除显式 `action_mode="tool_calling"`。
- 删除显式 `tools=ToolsConfig()`、`evaluation=EvaluationConfig(reproducible=False)`、
  `observability.langsmith_enabled=False`：直接省略，让 `AppConfig` 的
  `default_factory` 接管。
- 容器特化字段保留：`dataset.root_path=DEFAULT_INPUT_DIR`、
  `run.output_dir=DEFAULT_INTERNAL_RUNS_DIR`、
  `observability.gateway_caps_path=DEFAULT_GATEWAY_CAPS_PATH`。

**修改 `run()`**：

- 移除 `replace(config.run, ..., max_workers=..., task_timeout_seconds=...)`
  中对 `max_workers` 的“无脑覆盖”：仅当外部传入 `max_workers` 形参时覆写；
  否则保留 dataclass 默认。
- `task_timeout_seconds` 仅当 `llm is not None`（测试态）时改为 `0`，
  否则保留默认。
- 容器路径相关 `replace(...)` 保留。

### 4.2 `src/data_agent_langchain/config.py`

- `AgentConfig.model` 默认值由 `"gpt-4.1-mini"` 改为 `"qwen3.5-35b-a3b"`，
  与 KDD Cup 提交目标模型对齐；本地 / 容器在 `MODEL_NAME` env 未设时
  落到同一值。

  > **备选**：保留 `gpt-4.1-mini`，把 `qwen3.5-35b-a3b` 改写为容器路径
  > 下的显式 fallback。该备选与“单一源”原则有轻微冲突；若用户最终选择
  > 该方案，需回填本节并相应在 `submission.py` 留一处显式 fallback。

- `_AGENT_ENV_VAR_MAP` 与 `_overlay_agent_env_vars` 不变。
- `AgentConfig` docstring 追加一行说明：`model` / `api_base` / `api_key`
  受 `MODEL_*` env 兜底，本地与容器共享同一协议。

## §7 风险与回滚

1. **风险**：已有镜像 / 提交脚本依赖 `DABENCH_*` env。
   **回滚**：恢复 `_int_from_env` 与两行 env 读取即可，本次改动是单
   文件局部修改。
2. **风险**：`AgentConfig.model` 改为 `"qwen3.5-35b-a3b"` 影响某些只跑
   本地 OpenAI 的开发者。
   **缓解**：本地走 `MODEL_NAME` env 显式设置 `gpt-4.1-mini` 即可，
   YAML 显式值仍优先。
3. **风险**：删除测试用例被误判为弱化测试。
   **缓解**：每条删除用例必须有等价替换（见 §5），并在 PR 描述里逐条
   对账。
