## §3 v3 新引入问题（N1-N3）

### §3.1 N1（**Medium - 阻塞**）：§6 V7 中 `_fake_impl` 代码片段对 `TaskRunArtifacts` 构造错误

**v3 §6 V7 原文**：

```python
def _fake_impl(*, task_id, config, run_output_dir, llm, graph_mode):
    # 同 test_per_task_failure_does_not_block_other_tasks 模板
    return TaskRunArtifacts(
        task_id=task_id, succeeded=True,
        prediction_csv_path=None, run_dir=run_output_dir / task_id,
    )
```

**问题**：`TaskRunArtifacts` 实际定义于 `@.../src/data_agent_langchain/run/runner.py:55-63`：

```@c:/Users/18155/learn python/Agent/kddcup2026-data-agents-starter-kit-master/.worktrees/feature-improve-config-params/src/data_agent_langchain/run/runner.py:55-63
@dataclass(frozen=True, slots=True)
class TaskRunArtifacts:
    """单任务跑完之后的产物清单。"""
    task_id: str
    task_output_dir: Path
    prediction_csv_path: Path | None
    trace_path: Path
    succeeded: bool
    failure_reason: str | None
```

3 个具体 bug：

1. **`run_dir=` 字段不存在** —— 真实字段名是 `task_output_dir=`。
2. **缺必填字段 `trace_path`** —— `frozen=True, slots=True` 的 dataclass 不接受省略。
3. **缺必填字段 `failure_reason`** —— 同上（`succeeded=True` 时仍须传 `None`）。

按字面照抄到 `tests/test_submission_runtime.py` 后会立刻
`TypeError: TaskRunArtifacts.__init__() got unexpected keyword argument 'run_dir'`
或 `missing N required keyword-only arguments: 'task_output_dir', 'trace_path',
'failure_reason'`。

**v2 审查 §3 M5 给的版本**留了 `...  # 同 test_per_task_failure_does_not_block_other_tasks 模板`
（ellipsis 占位，让实施者直接对照 `tests/test_submission_runtime.py:259-277` 复制），
而 v3 把 ellipsis 替换成"伪具体"代码片段时引入了上述 3 个错误。**这是 v3 相对
v2 的一处回归**。

**修订建议**：把 §6 V7 的 `_fake_impl` 改为基于 `tests/test_submission_runtime.py:259-277`
的真实模板：

```python
def _fake_impl(*, task_id, config, run_output_dir, llm, graph_mode):
    # 模板来自 tests/test_submission_runtime.py:259-277
    task_output_dir = run_output_dir / task_id
    task_output_dir.mkdir(parents=True, exist_ok=True)
    prediction_csv = task_output_dir / "prediction.csv"
    with prediction_csv.open("w", newline="", encoding="utf-8") as handle:
        csv.writer(handle).writerows([["id"], [1]])
    trace_path = task_output_dir / "trace.json"
    trace_path.write_text("{}", encoding="utf-8")
    return TaskRunArtifacts(
        task_id=task_id,
        task_output_dir=task_output_dir,
        prediction_csv_path=prediction_csv,
        trace_path=trace_path,
        succeeded=True,
        failure_reason=None,
    )
```

并在 V7 函数顶部追加 `import csv` / `from data_agent_langchain.run.runner import TaskRunArtifacts`
两条 import（v3 当前 V7 骨架已有第二条 import，仅需补 `csv`）。
