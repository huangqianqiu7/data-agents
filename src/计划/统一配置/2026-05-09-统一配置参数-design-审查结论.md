"先删后写"的执行者看到"找不到要删的东西"，反过来怀疑设计本身。建议
合入前按本审查 §3 完成事实更正。

### §3.5 F5：§4.1 关于 `run()` 内 `max_workers` 的"无脑覆盖"描述失真

**设计原文**（§4.1 修改 `run()` 项）：

> 移除 `replace(config.run, ..., max_workers=..., task_timeout_seconds=...)`
> 中对 `max_workers` 的"无脑覆盖"：仅当外部传入 `max_workers` 形参时
> 覆写；否则保留 dataclass 默认。

**核查**：`@/Users/18155/learn python/Agent/kddcup2026-data-agents-starter-kit-master/.worktrees/feature-improve-config-params/src/data_agent_langchain/submission.py:433`
当前已经是

```python
max_workers=max_workers if max_workers is not None else config.run.max_workers,
```

——**已经是条件覆盖**，不是"无脑覆盖"。`config.run.max_workers` 在
`build_submission_config()` 中由 `_int_from_env("DABENCH_MAX_WORKERS",
DEFAULT_MAX_WORKERS)` 决定；删除这两个东西之后，`config.run.max_workers`
就会自然落到 `RunConfig()` 默认。**`run()` 这一行根本不需要改。**

**修订建议**：把 §4.1 该 bullet 改为：

> `run()` 不需要修改 `max_workers` / `task_timeout_seconds`
> 的覆盖逻辑：当前已是
> `max_workers if max_workers is not None else config.run.max_workers`
> 与 `0 if llm is not None else config.run.task_timeout_seconds`
> 的条件写法，删除 `_int_from_env(...)` 后行为自动变成"无外部
> 形参 → 走 `RunConfig` 默认"。仅需删除 `build_submission_config()`
> 内的两行 `_int_from_env` 调用与对应实参。

## §5 设计取舍（D1-D3）

### §5.1 D1（**重要**）：建议把 §4.2 备选方案升为主方案

**设计原文**（§4.2）：

> `AgentConfig.model` 默认值由 `"gpt-4.1-mini"` 改为 `"qwen3.5-35b-a3b"`，
> 与 KDD Cup 提交目标模型对齐。
>
> > **备选**：保留 `gpt-4.1-mini`，把 `qwen3.5-35b-a3b` 改写为容器路径
> > 下的显式 fallback。

**论据**（支持采纳备选 / 反对当前主方案）：

1. **YAGNI / 最小改动原则**：当前 `AgentConfig.model = "gpt-4.1-mini"`
   被多处文档与测试引用（O7 三处、`tests/test_phase5_config.py:196`），
   改默认值会引起一连串文档与测试调整，回滚成本远高于"容器路径多写
   一处显式 fallback"。
2. **"无 surprising 改动"原则**：默认值原本是 OpenAI 旗舰小模型，
   是项目历史定位；切到 `qwen3.5-35b-a3b` 会让本地开发者在不知情时
   把请求发去内部网关风格的模型名，反而更容易 surprise。
3. **`rules.zh.md` §5.4 LLM 身份必须经 env**：既然 LLM 身份字段
   "**必须**通过环境变量注入"，dataclass 默认值在生产路径上**永远
   不应该被使用**——它存在只是为了让 dev 在 YAML 也不写、env 也不
   设的纯离线开发态有可启动的占位值。这一占位值用什么名字与"评测
   官方默认模型"无关。
4. **"单一权威"语义**：保留 `gpt-4.1-mini` 作为默认 + 容器在
   `MODEL_NAME` 缺失时**显式 raise** `SubmissionConfigError`
   （而不是回退到 `qwen3.5-35b-a3b` 字面量），更彻底地落地"LLM 身份
   必经 env"，比"容器路径下的显式 fallback"还干净。

**修订建议（推荐）**：

> §4.2 重写：
> - `AgentConfig.model` 默认**保持** `"gpt-4.1-mini"`（与本地
>   OpenAI 开发态一致）。
> - `build_submission_config()` 中 `MODEL_NAME` 处理升级为：
>   - 空 → 抛 `SubmissionConfigError("Missing required environment
>     variable: MODEL_NAME")`（与 `MODEL_API_URL` 同级硬约束）；
>   - 非空 → 写入 `AgentConfig(model=...)`。
> - 同步删除 `DEFAULT_MODEL_NAME` 常量与 `__all__` 导出。
> - O7 三处文本因此自然不需要改。

如**仍坚持把默认改为 `qwen3.5-35b-a3b`**，请在 §7 风险分析里
显式承诺 O7 三处行号 + `tests/test_phase5_config.py:196` 全部更新。

### §5.2 D2：§6 缺字段级等价断言

设计 §3.1 把"唯一权威"作为一等公民，但 §6 验证清单只用 grep + 字段
点检三个常量（`max_workers` / `task_timeout_seconds` / `model`）。
更强的形式化等价应是：

> 新增 `tests/test_submission_config.py::test_submission_appconfig_equals_default_except_paths_and_identity`：
>
> ```python
> from dataclasses import replace
> from data_agent_langchain.config import default_app_config
>
> def test_submission_appconfig_equals_default_except_paths_and_identity(monkeypatch):
>     monkeypatch.setenv("MODEL_API_URL", "http://internal.model/v1")
>     cfg = build_submission_config()
>     baseline = default_app_config()
>     # 容器特化字段（路径 + LLM 身份）允许不同
>     normalized = replace(
>         cfg,
>         dataset=baseline.dataset,
>         agent=replace(cfg.agent, model=baseline.agent.model,
>                       api_base=baseline.agent.api_base,
>                       api_key=baseline.agent.api_key),
>         run=replace(cfg.run, output_dir=baseline.run.output_dir),
>         observability=replace(
>             cfg.observability,
>             gateway_caps_path=baseline.observability.gateway_caps_path,
>         ),
>     )
>     assert normalized == baseline
> ```

这条断言是"唯一权威"原则的形式化保险，未来若有人偷偷在
`build_submission_config` 加新字段差异，会立即 RED。
