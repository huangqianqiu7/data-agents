## §1 背景与目标

### 1.1 用户诉求

> 让 `data_agent_langchain` 项目除了 `api_key` 和对应的 URL 以外的其他
> 参数，在本地运行和容器运行都是同一套，所有参数配置都集中在同一个
> 地方。

### 1.2 评测规则约束

`rules.zh.md` §5.2 / §5.4：LLM URL / Key **必须**通过环境变量注入，不得
硬编码到 YAML 或镜像中。v2 在该约束基础上把 LLM **身份三字段**（URL /
Key / Name）一律提升为容器必填 env，把"非身份字段"全部下沉到
dataclass 默认。

### 1.3 目标

- `src/data_agent_langchain/config.py` 的 dataclass 默认值是**所有共享
  参数的唯一权威**。
- 本地（`load_app_config`）与容器（`build_submission_config`）对"非身
  份、非容器特化路径"的所有字段产出最终值**相等**。
- `submission.py` 不再包含可能与 dataclass 默认漂移的常量（`DEFAULT_*`
  三件套全删）。

## §3 设计原则

1. **唯一权威**：`config.py` dataclass 默认是所有共享参数的唯一权威。
2. **最小覆写**：`build_submission_config` 仅显式覆写"容器特化的 3 条
   路径"（`/input`、`/tmp/dabench-runs`、`/app/gateway_caps.yaml`）。
3. **LLM 身份必经 env**（v2 强化）：`MODEL_API_URL` / `MODEL_API_KEY` /
   `MODEL_NAME` 三者全部为容器路径**必填**；缺失统一抛
   `SubmissionConfigError`，错误信息只点名 env 名，不含字面值。
4. **本地路径继续走 env 兜底**：`_overlay_agent_env_vars` 三字段（local
   path）的"YAML > env > dataclass 默认"协议**保持不变**——本地纯
   离线开发态仍可不设任何 env 启动。
5. **测试等价替换**：删除任何测试用例必须给出等价替换；不允许静默
   弱化覆盖。

## §7 风险与回滚

1. **风险**：已有镜像 / 提交脚本依赖 `DABENCH_*` env，或调用方未设
   `MODEL_NAME`。
   **缓解**：评测系统的 starter kit 文档明确提示 `MODEL_*` 三件套必填；
   `DABENCH_*` 是项目内部调优开关，未公开宣传。
   **回滚剧本**：`git revert <commit>`；或局部
   `git checkout HEAD^ -- src/data_agent_langchain/submission.py
   src/data_agent_langchain/config.py` 恢复单文件，再补回测试改动。
2. **风险**：D1 的 `MODEL_NAME` 必填硬约束在 `tests/test_submission_runtime.py`
   引发 ~12 处 `setenv` 增补，量大易漏。
   **缓解**：用 conftest 级 fixture（如已有）统一 setup；TDD 顺序中
   先改测试再改代码，让 RED 失败立刻暴露漏改的测试。
3. **风险**：删除测试用例被误判为弱化测试。
   **缓解**：每条删除用例必须有等价替换（见 §5）；PR 描述里逐条对账。
